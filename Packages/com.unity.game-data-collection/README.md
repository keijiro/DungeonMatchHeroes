# Game Data Collection

A Unity Editor package that automatically captures version-controlled snapshots of your project during AI Assistant conversations. Each user or assistant message triggers a git commit, creating a detailed history of how the project evolves through AI-assisted development.

## Prerequisites

- **Unity 6000.3** or later
- **Git** installed and available in your system PATH
- **Git LFS** installed (`brew install git-lfs` on macOS, or download from https://git-lfs.com)
- **GitHub CLI** (optional, for creating remote repos from the terminal): `brew install gh`

Verify your installations:

```bash
git --version
git lfs --version
```

## Setting Up a Git Repository for Your Unity Project

If your Unity project is not already in a git repository, follow these steps. Run all commands from the **root folder** of your Unity project (the folder that contains `Assets/`, `ProjectSettings/`, etc.).

### 1. Initialize the Repository

```bash
cd /path/to/your/unity/project
git init
git lfs install
```

### 2. Configure .gitignore

Create a `.gitignore` file in the project root with the following contents. This excludes Unity-generated folders that should not be version-controlled (`Library/`, `Temp/`, `Obj/`, etc.) while preserving essential project files.

The `**/` prefix ensures these patterns match at any directory depth, which is important if your repo contains multiple Unity projects or the Unity project is not at the repo root.

```gitignore

**/[Tt]emp/
**/[Oo]bj/
**/[Bb]uild/
**/[Bb]uilds/
**/[Ll]ogs/

**/[Uu]ser[Ss]ettings/
!**/[Uu]ser[Ss]ettings/EditorUserSettings.asset

# MemoryCaptures can get excessive in size.
# They also could contain extremely sensitive data
[Mm]emory[Cc]aptures/

# Recordings can get excessive in size
[Rr]ecordings/
```

### 3. Set Up Git LFS

Git LFS is required for tracking binary and large files that are common in Unity projects (textures, audio, 3D models, etc.). Without it, your repository will quickly become bloated and may exceed GitHub's file size limits.

Run the following commands to configure LFS tracking:

```bash
# Textures and images
git lfs track "*.psd" "*.png" "*.jpg" "*.jpeg" "*.gif" "*.bmp" "*.tga" "*.tif" "*.tiff" "*.exr" "*.hdr"

# Audio
git lfs track "*.wav" "*.mp3" "*.ogg" "*.aif" "*.aiff"

# 3D models
git lfs track "*.fbx" "*.obj" "*.blend" "*.ma" "*.mb" "*.max" "*.dae"

# Video
git lfs track "*.mp4" "*.mov" "*.avi" "*.webm"

# Fonts
git lfs track "*.ttf" "*.otf"

# Unity serialized assets
git lfs track "*.asset" "*.prefab" "*.unity" "*.mat" "*.anim"
git lfs track "*.controller" "*.overrideController"
git lfs track "*.physicMaterial" "*.physicsMaterial2D"
git lfs track "*.lighting" "*.shadergraph" "*.shadersubgraph"
git lfs track "*.renderTexture" "*.terrainlayer" "*.mixer" "*.mask" "*.cubemap"

# Native plugins
git lfs track "*.dll" "*.so" "*.dylib" "*.a"
```

This creates a `.gitattributes` file. Make sure to commit it along with your project.

### 4. Create the Initial Commit

```bash
git add .gitignore .gitattributes
git add Assets/ ProjectSettings/ Packages/manifest.json
git commit -m "Initial commit"
```

### 5. Create a Remote Repository and Push

Using the GitHub CLI:

```bash
# Create a private repo and push
gh repo create YourProjectName --private --source . --push
```

Or manually:

1. Create a new repository on GitHub (or your preferred hosting).
2. Add the remote and push:

```bash
git remote add origin git@github.com:YourOrg/YourProjectName.git
git push -u origin main
```

## Installing the Package

### Option A: From the Internal Registry

Add the package to your project's `Packages/manifest.json`:

```json
{
  "dependencies": {
    "com.unity.game-data-collection": "1.0.0"
  },
  "scopedRegistries": [
    {
      "name": "Internal",
      "url": "https://artifactory.prd.cds.internal.unity3d.com/artifactory/api/npm/upm-candidates",
      "scopes": ["com.unity.game-data-collection"]
    }
  ]
}
```

Then open your Unity project and the package will be resolved automatically.

### Option B: From a Local Checkout

If you have a local clone of the `muse-editor` repository, add the package by path in `Packages/manifest.json`:

```json
{
  "dependencies": {
    "com.unity.game-data-collection": "file:/path/to/muse-editor/Packages/com.unity.game-data-collection"
  }
}
```

Or use the Unity Package Manager window:
1. Open **Window > Package Manager**.
2. Click the **+** button in the top-left corner.
3. Select **Add package from disk...**.
4. Navigate to the `com.unity.game-data-collection` folder and select its `package.json`.

## Enabling Auto Snapshot Recording

Once the package is installed and your project is in a git repository, automatic snapshot recording is enabled by default. Every time you send a message to the AI Assistant or receive a response, the package will:

1. Save all open scenes and assets.
2. Stage all project changes (`git add -A`).
3. Commit with a message containing the timestamp and a preview of the conversation message.
4. Optionally push to the remote repository.

### Using the Settings Window

Open the settings window from the Unity menu bar:

**AI Assistant > Internals > Data Collection > Snapshotting Settings**

This window provides the following controls:

| Control | Description |
|---|---|
| **Game Creation Recording** | Toggle automatic snapshots on/off. When enabled (default), every AI Assistant message triggers a commit. |
| **Branch Name** | Specify a git branch name to work on. Use the **Checkout** button to switch to an existing branch, or **Create** to create and switch to a new one. |
| **Auto Upload** | Toggle automatic push to remote after each snapshot. Enabled by default. |
| **Push to Origin** | Manually push all local commits to the remote repository. |
| **Take Snapshot** | Manually create a snapshot commit of the current project state. |
| **Rollback** | Revert the project to the previous snapshot. This performs a `git reset --hard` to the prior commit and reloads all scenes. |

### Settings Reference

| Setting | Persists Across | Default |
|---|---|---|
| Branch Name | Editor sessions (EditorPrefs) | Empty |
| Game Creation Recording | Current session only (SessionState) | `true` |
| Auto Upload | Current session only (SessionState) | `true` |
| Force Push | Current session only (SessionState) | `false` |

Note: **Game Creation Recording** and **Auto Upload** reset to their defaults each time Unity is restarted. If you want them disabled for a session, toggle them off after opening the editor.

## How It Works

1. **Hook Initialization** — On editor startup, the package hooks into the AI Assistant's `ConversationChanged` event via `AssistantEventHook`.
2. **Message Detection** — When a user or assistant message is marked as complete, a snapshot is triggered.
3. **Snapshot Mapping** — The message content, conversation ID, and message ID are recorded in `.ai/snapshot_mapping.json` at the project root.
4. **Git Commit** — All project changes are staged and committed. The commit message includes the role (User/Assistant), a timestamp, and the first 15 words of the message content for easy identification:
   ```
   User Message Snapshot 2026-03-16_14-30-00: can you build a room with a table and two chairs...
   Assistant Message Snapshot 2026-03-16_14-30-15: I have created a room with a wooden table and...
   ```
5. **Auto Upload** — If enabled, the commit is pushed to the remote repository automatically.

## Troubleshooting

**"git: command not found" errors in the Unity console**
Unity-spawned processes may not inherit your shell's PATH. On macOS, the package automatically prepends `/opt/homebrew/bin` and `/usr/local/bin` to the PATH. If git is installed elsewhere, ensure it is accessible via one of these paths or symlink it.

**Git LFS errors on push**
Make sure `git lfs install` has been run in your repository and that `.gitattributes` is committed. If large files were committed before LFS was configured, you may need to use `git lfs migrate` to rewrite history.

**No snapshots are being created**
- Verify the AI Assistant window is open (**Window > AI > AI Assistant**).
- Check that **Game Creation Recording** is enabled in **AI Assistant > Internals > Data Collection > Snapshotting Settings**.
- Look for errors in the Unity console with the prefix `[GameDataCollection]`.

**Auto Upload fails**
- Ensure the remote is configured: `git remote -v`.
- Ensure you have push access to the remote repository.
- Check the Unity console for detailed error messages.
